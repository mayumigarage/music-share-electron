1. Render へのデプロイ方法

現在のコードベースで Render にデプロイするには、サーバー側の準備とクライアント側の接続先変更が必要です。

サーバー側（src/server/）

server.ts は既に process.env.PORT を読み込むようになっているので、Render の動的ポート割り当てに対応しています。

const PORT = process.env.PORT ? parseInt(process.env.PORT, 10) : 5000;

ただし、package.json の start は Electron 用なので、Render 用の起動スクリプトを追加してください：

{
  "scripts": {
    "start": "...", // Electron 用（既存）
    "server": "tsc -p tsconfig.server.json && node dist/server/server/server.js",
    "server:prod": "node dist/server/server/server.js",
    "build:server": "tsc -p tsconfig.server.json"
  }
}

2. ゲスト側の接続方法

アプリ起動後、ゲストがルームに参加する手順と内部処理は以下の通りです。

UI 操作手順

1. モーダルを開く
アプリ起動時に表示されるルーム作成/参加モーダル（room-modal.ts）で、「参加する」タブを選択します。
2. 情報を入力
  - ルームID: ホストに教えてもらったID（例：873-921）を入力
  - ユーザー名: 自分の表示名を入力
3. 参加ボタンを押す
wsClient.joinRoom(roomId, userName) が呼ばれ、サーバーに JoinRoom イベントが送信されます。

内部処理フロー

ゲスト: joinRoom(roomId, userName)
  → Socket.IO サーバー: JoinRoom イベント
    → room-manager.ts: ルーム存在確認 → メンバー追加
    → ホストに: UserJoined 通知
    → ゲストに: RoomJoined 通知（room + user オブジェクト）
      → UI: 右パネルにメンバーリスト表示
      → UI: キュー・履歴・プレイヤー状態を同期

WebRTC P2P 接続（HostBroadcast モード時）

ゲストが音声を受信する場合、ホストとの WebRTC 接続が確立されます：

1. ホストが getDisplayMedia() で音声をキャプチャ
2. ホストが各ゲストに対して RTCPeerConnection を作成し、SDPOffer を送信（Socket.IO 経由）
3. ゲストが SDPAnswer を返信
4. ICE Candidate を交換し、P2P 接続確立
5. 音声ストリームがサーバーを経由せず、ホスト → ゲストに直接流れる

▎ サーバーは シグナリング（SDP/ICE の中継）のみを行い、音声バイナリは一切転送しません。

補足：同時接続数の制限

第一フェーズではホスト1人に対してゲスト3〜5人までを推奨としています（RoomModal では maxGuests = 5 がデフォルト）。超過時はサーバー側で参加拒否または警告を返す実装が必要です。
